# Stage 2: Refined Agent + Workflow Usage

> **Note:** For this course offering, Stage 2 and the final proposal are submitted together. See [SUBMISSION_CHECKLIST.md](SUBMISSION_CHECKLIST.md) and the repository root [workflow_usage.md](../workflow_usage.md).

Stage 2 evaluates whether you refined the Stage 1 workflow or agent and used it to generate, revise, or evaluate proposal content.

## Due Date

Stage 2 is due Friday, June 12, 2026, 11:59 PM Pacific Time.

Late submissions are accepted until Sunday, June 14, 2026, 11:59 PM Pacific Time with a 20% penalty.

## Required Deliverables

Submit:

- refined agent implementation or reproducible workflow artifact
- `workflow_usage.md`
- run transcript, screenshots, logs, or demo
- `AI_USAGE.md`

## Workflow Usage Evidence

Your submission should make it possible to see how proposal content was produced or improved through your workflow. Include enough run evidence for graders to reconstruct at least one meaningful agent-student interaction and one revision or evaluation step.

Use the Stage 2 section of [grading_rubric.md](grading_rubric.md) for detailed scoring expectations.

## Important Rule

If you only use vibe coding or a chat model to directly generate a final proposal, you can still receive Stage 3 proposal-quality credit. However, you will receive little or no Stage 2 credit unless you show that your own workflow or agent guided proposal creation or revision.

## Stage 2 Checklist

- Refined agent or workflow is demonstrated.
- Workflow usage is documented.
- Run evidence shows student-agent interaction.
- Run evidence shows that the workflow checks proposal coverage or quality.
- Run evidence shows that the workflow identifies weaknesses or revision priorities.
- AI usage log records tools, prompts, model calls, and human edits.
- At least one revision loop is shown.
